# Security Policy

## Reporting a Vulnerability

If you believe you have discovered a security vulnerability in Zen, please do not open a public GitHub issue.

Instead, report it privately by contacting the maintainer through GitHub or by using the contact information provided in the repository profile.

Please include:

* A description of the vulnerability.
* Steps to reproduce the issue.
* The affected Zen version.
* The affected Go version.
* Any proof-of-concept code, if applicable.

Every report will be reviewed as quickly as possible.

---

## Supported Versions

As Zen is currently in the early stages of development, only the latest released version is supported with security fixes.

| Version        | Supported |
| -------------- | --------- |
| Latest release | ✅         |
| Older releases | ❌         |

---

## Security Updates

Security fixes will be released as patch versions whenever practical and documented in the project's changelog.
